## 概要

のもとけ型に寄せて、カテゴリは粗く保ち、選手タグと話題タグで深く回遊させる。
ホーム / 記事末 / タグ一覧のどこで、読者が次の関連記事へ飛ぶかを固定する。
正本要件は `docs/handoff/giants_agile_requirements.md` §6 / §9。親チケットは `doc/014` (`2d92405`) / `doc/017`。

## 決定事項

### 回遊の基本

- 親カテゴリは増やさない:
  `doc/014` の `ALLOWED_CATEGORIES` 4 件を固定し、細分類はカテゴリではなくタグで吸収する。新 category 案は初期話題タグへ寄せ、追加は user 判断に留める。
- 選手タグ:
  巨人の主役選手は記事ごとに 1 件以上を必須付与し、同一選手の過去記事へ直接返す。記事末 / ホーム / タグ一覧の 3 箇所で回遊の中心に固定する。
- 話題タグ:
  初期集合は `ファーム / 公示 / 契約 / 野球データ / 予告先発 / 番組 / 試合前 / 試合後 / プロ野球` とする。集合外の新規話題タグは `doc/015` の observation ticket 経由で審査する。

### 導線位置

- ホーム:
  `doc/017` の「今日の試合」ハブ直下に主要選手タグ群(5〜10 選手)を置き、中段に主要話題タグ群、下部に全タグ一覧への導線を置く。
- 記事末:
  第 1 優先は同一選手タグの最新 3 本、第 2 優先は最大 weight の話題タグ一致の最新 3 本、第 3 優先は同一 category の最新 2 本とする。重複除外で計 5〜8 本を目安に返す。
- タグ一覧:
  巨人主要選手タグを最上部、主要話題タグを次段、非主要タグを五十音順またはアルファベット順で下段に置く。引退選手などのアーカイブタグは別枠に分ける。

### 非機能

- タグ管理:
  A7 `tag_category_guard` の `TAG_TARGET_LOW=15` / `TAG_MAX=20` を崩さず、初期タグ集合(選手 + 話題)を台帳管理する。重複タグや表記揺れは A6 `source_id` 正規化に準拠して統合する。
- 不可触範囲:
  固定版レーン / AIエージェント版ともに本便は doc のみとし、既存 mail / automation、`quality-monitor` / `quality-gmail` / `draft-body-editor`、Gmail / SMTP / sendmail / scheduler / env / secret には触らない。published write や WP write、タグ一覧 UI / 記事末 UI の変更は後続フロント便に分離する。
