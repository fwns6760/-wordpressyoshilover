## 採番ノート

257〜259 は QA / RC / T1 派生枠として空け、260-MKT は攻め記事型・テンプレ設計のマーケ枠として採番する。

## 状態

## gap 3: T2 との関係 (重要)

監督・コーチ・選手コメント要点 (記事型 F) は、**260-MKT で別実装にしない**。

方針:
- T2 (262-QA-title-subject-name-required の延長) = 既存 manager / coach_comment / player_comment のタイトル・本文テンプレ改善
- 260-MKT = T2 で整えたコメント記事を、将来の独自記事型・観戦ガイド・話題導線にどう活用するかの設計
- 260-MKT で新たにコメント系の自動生成を増やさない
- 260-MKT では、コメント記事の「使い方」「導線」「テンプレ原則」を定義する

T2 で解決すべきもの (本 ticket の前提):
- 人名なしタイトル
- 「コメント整理」系の汎用タイトル
- 「ベンチ関連の発言ポイント」系の generic suffix
- speaker_name に「チーム」「大敗」など non-name が入る問題
- source にない一般論 padding
- quote_count=0 で薄い記事が出る問題

260-MKT でやること (本 ticket scope):
- T2 後のコメント記事を観戦ガイドや同 game 関連記事にどう見せるか
- どのコメント記事が「攻め記事型」として使えるか
- 手動 pilot でどのテンプレが読めるかを確認する

## gap 5: 法務 / 著作権境界

ルール:
- テレビ発言は長文書き起こし禁止
- 公式動画の長文引用禁止
- X 投稿本文の無断転載禁止
- 5ch レス本文の転載禁止
- マスコミ X は原則 oEmbed またはリンク参照
- SNS / 掲示板は話題検知のみ
- 記事本文の根拠は公式 / スポーツ紙 / 番組 / 公式動画で確認できる情報に寄せる
- 著作権・引用リスクが高い場合は手動確認に倒す

---

## gap 6: KPI / 試行期間

初期案:
- 各記事型を手動または半自動で 3〜5 本試す
- 試行期間は 1〜2 週間
- 自動化判断はその後

評価指標:
- ユーザー手直しが少ない
- タイトルが自然
- 本文に一般論 padding がない
- source にない展望がない
- 読者向け価値がある
- X 手動投稿に使える
- 同 game 関連記事や観戦ガイドに接続しやすい
- コストが増えていない

成功条件:
- 手直し率が低い
- generic title が出ない
- 記事型として継続可能
- 事実ミスがない
- ユーザーが「この型は使える」と判断できる

## gap 9: noindex / index 仮方針

260-MKT の記事型は、最初は **default noindex 維持**。

理由:
- まだ記事型の品質検証段階
- 251-SEO 解放前の前提条件群が揃っていない (canonical drift / duplicate cleanup / title 品質安定 等)

仮方針:
- 初期 pilot は default noindex
- 手直しが少ない
- 事実確認が強い
- タイトルが明確
- 重複していない
- 読者価値がある
- ユーザーが index 候補と判断したものだけ、後で 251-SEO の index 解放戦略に接続

260-MKT 内で noindex 解放はしない。

## テンプレ例 (3 型)

### A. テレビ・公式動画発言メモ

タイトル型:
```
{発言者名}が語った{テーマ}　巨人ファンが確認したいポイント
```

本文型:
```
【発言の内容】
{番組名/公式動画/媒体名}で、{発言者名}が{テーマ}について触れました。

【確認できること】
- {fact_1}
- {fact_2}

【巨人ファン向けメモ】
この話題は、{試合/選手/起用/二軍/復帰}を見るうえでの材料になります。
```

### B. 小型データ記事

タイトル型:
```
巨人スタメン変更メモ　{選手名}が{変化内容}
```

本文型:
```
【変更点】
前回から変わったのは{変更内容}です。

【確認できるデータ】
- {打順}
- {守備位置}
- {出場継続}
- {相手投手}

【見るポイント】
今日は{選手名}の{役割/打順/守備位置}を確認したいところです。
```

### C. X/5ch 話題トリガー記事

タイトル型:
```
巨人ファンが気になった{話題}　確認できる情報は
```

本文型:
```
【話題になっている点】
{話題}について、巨人ファンの間で注目されています。

【確認できる情報】
公式/スポーツ紙/番組で確認できる内容は{fact}です。

【ここから見る】
推測ではなく、まずは{試合/公示/起用/選手状態}を確認していきます。
```

## 261-MKT-PILOT の予約

### 261-MKT-PILOT

pilot 候補 (初期 3 型):

#### 261 起動条件 (定量化)

以下のどれかを満たしたら、Claude が自律的に 261-MKT-PILOT の GO 提案を出す。

- RC 反映後、generic title (「コメント整理」「発言ポイント」「ベンチ関連」「実戦で何を見せ」「ベンチの狙いはどこ」「次のスタメンはどう」)が **24h で 5 件以下**まで明確に減った

- 262-QA 対象 pattern (主語抜け / 助詞始まり / generic noun のみ / 関連情報 escape) が **24h で 5 件以下**まで明確に減った
- ユーザーのタイトル/本文手直しが減ったと user が主観報告
- 260-MKT のテンプレ 6 型が完成 (本 ticket 完了)
- ユーザーが「攻め記事を試す」と明示
- 直近 24h で使える素材が十分あると確認できた:
  - 試合中以外の独自素材 (テレビ発言 / 公式動画 / 小型データ / X 話題トリガー) が **24h で 10 件以上** fetch 経路で観測されている (rss_fetcher の skip_reason / source mix の集計から定量判定)

#### 261 動作仕様

- 完全自動化しない (3〜5 本の手動/半自動 pilot で評価)

- X 投稿は user 手動判断

- 評価指標: KPI 項目 (gap 6 参照)
- 成功条件達成 → 段階拡大判断 / 自動化判断
