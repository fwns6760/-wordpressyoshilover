# 失敗パターン（再発防止）

## 失敗1: 新規Claudeチャット経由でenv変更 【最重要】

### 何が起きたか
2026-04-17。よしひろさんが新規Claudeチャットを開き、`RUN_DRAFT_ONLY=0` を設定した。
既存Claudeチャット（Claude Opus）の文脈なしに変更が実行された。

### 結果
4記事がpublishされた。意図しない公開。

### 実際の影響
- 不完全な品質の記事が一時的に公開URL状態になった
- 4記事をdraft戻しするためのCloud Run操作が必要になった
- よしひろさんの体力を大幅に消耗した

### 根本原因
- 新規チャットはプロジェクトの文脈（「今はdraft観察フェーズ」）を知らない
- 「RUN_DRAFT_ONLY=0」の変更が何を意味するか理解せずに実行された

### 対策（現在のルール）
1. **env変更は必ず既存進行中Claudeチャット経由**
2. 新規チャットでは絶対にenvを触らない
3. Cloud ConsoleのGUIからも変更しない（Claudeのチャット経由を通す）
4. 変更前に「今の全env」をClaudeに確認させる
5. 変更後は必ずsmoke testを実行する

### 事故後の処置結果（2026-04-18朝 Claude Code実測確認）

確認方法: WP REST API `GET /wp-json/wp/v2/posts/{id}` + 公開URLへのHTTPアクセス

| post_id | タイトル（先頭） | status | 公開URL |
|---------|--------------|--------|--------|
| 62483 | 吉川尚輝が２軍戦で１安打１盗塁... | `draft` ✅ | 404 ✅ |
| 62486 | ルシアーノ、１軍の練習に合流... | `draft` ✅ | 404 ✅ |
| 62489 | 巨人スタメン 巨人 vs ヤクルト... | `draft` ✅ | 404 ✅ |
| 62493 | 巨人スタメン ヤクルト戦（神宮）... | `draft` ✅ | 404 ✅ |

**全4記事: draft戻し完了確認済み。公開URL 4件とも 404。**

## 失敗2: ゴールデンテスト無確認受け入れ

### 何が起きたか
2026-04-17午前。title collision修正の実装後、よしひろさんが「OKそう」と感覚的に受け入れた。
その後、別の修正でゴールデンテスト（pytest）が失敗していることが判明した。

### 結果
- テストが落ちた状態でrevisionが積み重なった
- Codexが修正するたびにテストを確認させる手間が発生

### 対策（現在のルール）
1. Codexのすべての実装後に `pytest --tb=short -q` を実行
2. テスト354本グリーンが受け入れ条件
3. よしひろさんの受け入れ前にCodexがテスト結果を報告する

## 失敗3: 体力減らしモード違反（選択肢の乱発）

### 何が起きたか
Claudeが「A案とB案があります。どちらにしますか？」という提示を複数回行った。
よしひろさんが「体力消耗」「疲れる」と反応した。

### 結果
- よしひろさんの判断コストが増大
- 作業効率が低下
- 「体力減らしモード」のルール化につながった

### 対策（現在のルール）
1. 選択肢は1つに絞って推奨する
2. 「AとBどちら？」より「Aを推奨。差し戻すならBも可」
3. 指示書はCodexに一括投げ形式で作成する
4. よしひろさんに聞くのは「合格か差し戻しか」の2択のみ

## 失敗4: live_update の general 漏れ

### 何が起きたか
2026-04-17夕方。`live_update` subtypeが `publish` 側で `general` として扱われ、
publish可能状態になっていた。

### 結果
- `live_update` が公開される可能性があった（未然に防止）
- 🔴緊急修正として revision 00127-98x で対応

### 対策（現在のルール）
1. `live_update` は `ENABLE_PUBLISH_FOR_LIVE_UPDATE=0` で常時禁止
2. subtype分類ロジックに `live_update` の特別ケースを明示
3. 受け入れチェックリストに「live_updateが公開されていないこと」を追加

---

## 失敗5: featured_media観測漏れ

### 何が起きたか
2026-04-17夕方。`draft-only` モード中に `featured_media_missing` イベントの観測が
低下していた（ログに出なくなっていた）。

### 結果
- draft品質の問題を観察できない時期があった
- 🔴緊急修正で `featured_media_observation_missing` ログを追加

### 対策（現在のルール）
1. `draft-only` 中も画像欠落を観察するログを維持
2. `featured_media_observation_missing` イベントを品質監視に含める
