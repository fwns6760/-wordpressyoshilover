## Xポストの型（ヨシラバーで使う3パターン）

### 速報型（試合前・試合中）
```
【今日のスタメン発表】
1番 〇〇
2番 〇〇
...
先発: 〇〇

#巨人 #読売ジャイアンツ #スタメン
```
- シンプル・事実ベース
- ハッシュタグは3つまで

### 解説型（試合後・分析）
```
今日の〇〇の投球、データで振り返る⚾

防御率〇〇→本日〇回〇失点
奪三振率〇〇→本日〇K

スタッツの上振れか実力か…次登板に注目👀

#巨人 #〇〇投手名 #読売ジャイアンツ
```
- データを使うと反応が良い
- 疑問形で終わると返信が来やすい

### 感情型（勝利・好プレー）
```
〇〇のホームラン最高すぎた！！！🎊
あの場面で打てるのが〇〇の凄さ✨

今日もありがとう！！

#巨人 #〇〇選手名 #読売ジャイアンツ
```
- 感情を出す
- ファン目線で書く
- 過度な批判はしない

## 自動生成X文面（AI生成）のチェックポイント

`X_POST_AI_MODE=gemini` で生成されるX文面の受け入れ基準:

### OK
- 280文字以内
- 自然な日本語
- ハッシュタグが適切（2〜4個）
- 巨人ファン視点で書かれている
- 事実と一致している

### NG（差し戻し）
- 敵対的・批判的な内容
- 事実誤認（スコア・選手名の誤り）
- 不自然な言い回し（AI感が出ている）
- 280文字超過
- ハッシュタグが多すぎる（5個以上）

### 観察ログで確認すること
```bash
# AI生成X文面の確認
gcloud logging read \
  'resource.type="cloud_run_revision" AND jsonPayload.event="x_post_ai_generated"' \
  --project baseballsite --limit=10 --format=json
```

## AUTO_TWEET設定の確認

X自動投稿が有効になる条件:
```
AUTO_TWEET_ENABLED=1
AUTO_TWEET_CATEGORIES=試合速報,選手情報,首脳陣,ドラフト・育成
AUTO_TWEET_REQUIRE_IMAGE=1  ← 画像なしはX投稿しない
X_POST_AI_MODE=gemini
ENABLE_X_POST_FOR_{subtype}=1  ← 各カテゴリのフラグ
```
