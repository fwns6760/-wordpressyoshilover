### 2. 【T-001 🔴 調査のみ】WP REST API の `?status=draft` フィルタが機能していない

**事実**:
- `curl "<WP>/wp-json/wp/v2/posts?status=draft&per_page=100&context=edit"` で
  返ってくる36件が全て `status=publish`
- `?status=publish` / `?status=any` でも同じ36件
- 単独 GET（例 `posts/62483`）では `status=draft` が正しく返る
  → DB側は正常、list API でフィルタが効いていない

**やってほしいこと**:
1. WP の `functions.php` / mu-plugins / 有効プラグインに、
   `pre_get_posts` / `rest_post_query` / `rest_{post_type}_query` /
   `rest_prepare_post` 系のフィルタが登録されていないか調査
2. 特に list API から draft を除外するような処理がないか
3. 認証（Basic Auth のユーザー権限）が list API で draft を見る権限を持っているか確認

**報告形式**: 原因と修正方針（修正実装は別依頼）
**制約**: 調査のみ。WP 側の設定変更・deploy はしない。
