## Step A. 【T-002】p=61981 の opponent 阪神 → 楽天 を修正

### 目的

公開中の postgame 記事「巨人阪神戦 試合の流れを分けたポイント」を、
実試合の対戦相手（楽天）に合うよう修正する。evidence は nikkansports 2026-04-14 記事。

### 実行コマンド（完全形）

```bash
cd /home/fwns6/code/wordpressyoshilover
set -a; source .env; set +a

# 1. 現状を取得して diff の下書きを作る（dry-run）
python3 <<'PY'
import os
from src.wp_client import WPClient

wp = WPClient(
    url=os.environ["WP_URL"],
    user=os.environ["WP_USER"],
    password=os.environ["WP_APP_PASSWORD"],
)
post = wp.get_post(61981)
title = post["title"]["raw"]
content = post["content"]["raw"]
print("=== TITLE BEFORE ===")
print(title)
print("=== TITLE AFTER (preview) ===")
print(title.replace("阪神", "楽天"))
print("=== CONTENT '阪神' occurrences ===")
for i, line in enumerate(content.splitlines()):
    if "阪神" in line:
        print(f"L{i}: {line}")
PY
```

上の出力を **必ず目視確認** してから、以下の apply を実行:

```bash
python3 <<'PY'
import os
from src.wp_client import WPClient

wp = WPClient(
    url=os.environ["WP_URL"],
    user=os.environ["WP_USER"],
    password=os.environ["WP_APP_PASSWORD"],
)
post = wp.get_post(61981)
new_title = post["title"]["raw"].replace("阪神", "楽天")
new_content = post["content"]["raw"].replace("阪神", "楽天")
wp.update_post_fields(61981, title=new_title, content=new_content)
print("updated 61981")
PY

# 2. 修正後の fact_check
python3 -m src.acceptance_fact_check --post-id 61981 --json | python3 -m json.tool | head -40
```

### 期待される出力

- dry-run 側: title と content に「阪神」を含む行がリストアップされる
- apply 側: `updated 61981` の1行
- fact_check 側: `result: "green"` かつ `findings: []`

### 成功判定の基準

- `acceptance_fact_check --post-id 61981` の出力で:
  - `result` が `"green"`
  - `findings` が空配列 `[]`
  - 特に `opponent` の findings が消えている

### 失敗時の対応

- 置換が意図しない文脈（例: 阪神出身の選手名、他チームとの対戦履歴の言及等）を巻き込んでしまった場合:
  ```bash
  # 事前に控えた old_title / old_content に戻す
  python3 <<'PY'
  import os
  from src.wp_client import WPClient
  wp = WPClient(url=os.environ["WP_URL"], user=os.environ["WP_USER"], password=os.environ["WP_APP_PASSWORD"])
  wp.update_post_fields(61981, title=OLD_TITLE, content=OLD_CONTENT)
  PY
  ```
- apply 前に `old_title` / `old_content` を変数に保持し、rollback できる状態で実行すること

### 絶対にやらないこと

- `status=publish` を `draft` に変えない（`update_post_status` は呼ばない）
- featured_media / categories / tags を変えない
- content のうち、阪神に関係のない編集は一切しない
- `ENABLE_PUBLISH_FOR_*` / `RUN_DRAFT_ONLY` は触らない
