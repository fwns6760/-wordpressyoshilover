### 目的

T-010 対象 19件を以下 3 カテゴリに分類し、共通原因・修正方針・工数を整理する。

分類カテゴリ（想定）:
- **(A) 旧記事形式**: source block を埋め込む前の世代の記事で、構造化された参照ブロック自体が存在しない
- **(B) source block 欠落**: 本文中に URL 文字列は存在するが、構造化ブロックに入っていない（裸 URL / 画像キャプション / テキスト中の URL）
- **(C) 抽出漏れ**: 構造化ブロックは存在するが、現在の `extract_source_links()` が認識できないフォーマット

### 要件

1. **対象 19 件**:
   61947 / 61946 / 61939 / 61816 / 61812 / 61799 / 61779 / 61776 / 61773 / 61770 / 61754 / 61596 / 61572 / 61598 / 61600 / 61576 / 61574 / 61556 / 61554

2. **各記事について実施すること**:
   - `WPClient.get_post(post_id)` で content 取得
   - 本文を読み、URL（`https://` / `http://`）の出現箇所・形式を洗い出す
   - `extract_source_links()` を実際に呼んで何が拾えて何が拾えていないかを比較
   - 上記 3 カテゴリのどれに当たるか分類

3. **集計**:
   - カテゴリ別件数（例: A=10件 / B=7件 / C=2件）
   - 各カテゴリ内の代表 post_id を最大 3 件ずつ
   - `extract_source_links()` が認識できないフォーマット例を具体的に列挙（HTML スニペット）

4. **修正案比較**:
   以下 3 案について、**どの案が何件カバーできるか**・**副作用リスク**・**工数見積り** を整理:
   - (a) `extract_source_links()` の抽出パターン拡張
     - 本文末尾の「参考:」「出典:」「source:」ラベル付きテキスト
     - 裸の URL（既知ドメイン: nikkansports.com / sponichi.co.jp / hochi.news / sanspo.com / baseballking.jp 等）
     - 画像キャプション内リンク
   - (b) 記事側を後追いで改修（source block を手動 or バッチで補填）
     - 対象件数（どの記事が補填可能か）
     - 自動バッチ化の可否
   - (c) 「古い記事は source_reference 検査対象外」の除外ルール（yellow を抑制）
     - 閾値（post date / modified date / 記事 ID の境目）をどこに置けるか

5. **推奨方針**:
   調査結果をふまえて、「本来はこの順序で攻めるべき」という推奨を 1 本に絞る
