## 依頼内容

### 目的

`src/draft_audit.py` の `extract_source_links()` に以下 4 パターンの認識を追加し、publish 19件中 **最低 12件** で `source_reference_missing` yellow を解消する。

### 要件

1. **対象ファイル**:
   - `src/draft_audit.py` の `extract_source_links()`
   - 必要なら `src/acceptance_fact_check.py` の `_source_reference_facts()` 側の呼び出しも合わせて調整
   - `tests/test_draft_audit.py`（無ければ新設）に regression test を追加

2. **追加する抽出パターン 4 種**:
   - **(P1) `【引用元】` 見出し + 次段落アンカー**（C 対象、2件）
     ```html
     <h3 class="wp-block-heading">【引用元】</h3>
     <p><a href="https://www.nikkansports.com/...">日刊スポーツ「...」</a></p>
     ```
   - **(P2) `引用元:` / `出典:` / `参考:` ラベル付き段落**（B 主力、多数）
     ```html
     <p style="font-size:13px;color:#666;">
       2026年4月13日 現在 更新 / 引用元:
       <a href="https://npb.jp/...">NPB 個人打撃成績（...）</a>
     </p>
     ```
     ```html
     <p style="font-size:0.8em;color:#999;">
       出典:
       <a href="https://www.nikkansports.com/...">https://www.nikkansports.com/...</a>
     </p>
     ```
   - **(P3) 本文末尾の小文字 footer リンク**（B の一部）
     - `<p>` のスタイル属性に `font-size:0.8em` / `color:#666` / `color:#999` などが付いた末尾段落のアンカー
   - **(P4) `[[1]](url)` 形式**（B 単発）
     ```html
     ... [[1]](https://www.nikkansports.com/...) ...
     ```

3. **副作用防止**:
   - 本文中の unrelated な外部リンク（Twitter 埋め込みの tweet_url、広告、関連記事ウィジェット等）を **source として誤抽出しない**
   - 既存の `📰 参照元:` 認識が壊れないこと

4. **regression test 追加**（`tests/test_draft_audit.py`）:
   - P1〜P4 それぞれ 1 本ずつ、合計 **4 本以上** の新規テスト
   - 各テストは HTML スニペットを渡して期待 URL が返ることを確認
   - ネガティブケース（Twitter 埋め込みを source として拾わない）も 1 本

5. **受け入れ条件**:
   - `python3 -m unittest discover -s tests` が **全 pass**
   - `acceptance_fact_check` 再実行:
     - T-010 対象 19 件のうち **最低 12 件** で `source_reference_missing` yellow が解消
     - A=5 件（61754 / 61596 / 61598 / 61572 / 61600）は yellow 継続でよい
   - regression: p=62527 / p=61981 が **green 維持**
   - publish 再監査で **red が新規発生していない**
