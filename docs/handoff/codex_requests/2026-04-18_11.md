## 依頼内容

Cloud Run `yoshilover-fetcher` に最新 master を deploy。

### 目的

- Cloud Run を master HEAD (`a08d875`) に更新
- 次の Scheduler 発火から T-010 (a) 改修が効くようにする

### 要件

1. master HEAD から source deploy（第8便と同手順）
   - `docs/phase_c_runbook.md` の deploy 節に従う
   - local の未 commit 差分を含めない（`git archive HEAD` を一時ディレクトリへ展開）
2. **env 変数は現状維持**（変更しない）:
   - `RUN_DRAFT_ONLY=1`
   - `AUTO_TWEET_ENABLED=0`
   - `PUBLISH_REQUIRE_IMAGE=1`
   - `ENABLE_PUBLISH_FOR_*=0`
3. 新 revision の番号（`00133-*` 以降）を控える
4. smoke test:
   ```bash
   bash scripts/cloud_run_smoke_test.sh <new-revision>
   ```
5. Scheduler を 1 回だけ手動 trigger（任意の 1 job、例: `giants-weekend-pre`）
6. Cloud Logging で新 revision が起動し `draft_only=true`, `error_count=0` を確認

### 実行コマンド例（第8便と同じ）

```bash
tmp=$(mktemp -d)
git archive HEAD | tar -x -C "$tmp"
gcloud run deploy yoshilover-fetcher \
  --source "$tmp" \
  --region asia-northeast1 \
  --project baseballsite

bash scripts/cloud_run_smoke_test.sh <new-revision>

gcloud scheduler jobs run giants-weekend-pre \
  --location asia-northeast1 --project baseballsite

gcloud logging read \
  'resource.type=cloud_run_revision AND jsonPayload.event="rss_fetcher_run_summary"' \
  --limit 1 --project baseballsite --format json
```

### 成功判定

- new revision が `00133-*` 以降で ready
- traffic 100% が new revision
- env が上記4変数すべて期待値
- `/run` 実行で `draft_only=true`, `error_count=0`
- `severity>=ERROR` on new revision: **0 件**

### 失敗時

- smoke test 異常 or ERROR 発生 → 即 rollback:
  ```bash
  gcloud run services update-traffic yoshilover-fetcher \
    --to-revisions=yoshilover-fetcher-00132-lgv=100 \
    --region asia-northeast1 --project baseballsite
  ```
- rollback 後 `codex_responses/2026-04-18_11.md` に状態を記録して中止

### 禁止

- env 変更（`RUN_DRAFT_ONLY` / `ENABLE_PUBLISH_FOR_*` など）
- Secret Manager のバージョン変更
- Scheduler のスケジュール変更
- Phase C 公開解放（`ENABLE_PUBLISH_FOR_POSTGAME=1` など）
- コード変更（本便は deploy のみ）
