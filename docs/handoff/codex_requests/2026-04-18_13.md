## 依頼内容

### 目的

- 5件それぞれの元ネタ source URL を **タイトル文字列 + 記事内容** から特定
- 特定できた記事には **source block を WP 記事に追記**（第10便で拡張された P2 パターン `出典: <a>...</a>` を使う）
- 特定できなかった記事は **見送り**としてレポートに記録（除外ルール(c) 検討用）

### 要件

1. **source URL 特定手順**:
   - 各記事の title と本文冒頭 200字を使って、以下を順に試す:
     - nikkansports.com / sanspo.com / sponichi.co.jp / hochi.news / baseballking.jp のいずれかに類似記事がないか
     - 既存の draft_audit ログ（Cloud Logging）に同じ title で fetch された source URL が残っていないか
     - RSS fetcher の過去 item で該当 entry がないか（`docs/` の fetch 実績ログがあれば参照）
   - 見つかった URL は **記事日付と内容が整合することを確認**（タイトル一致だけで採用しない）

2. **source block 追記**（特定できた記事のみ）:
   - 本文末尾に以下の HTML ブロックを追加:
     ```html
     <p style="font-size:0.8em;color:#999;margin-top:16px;">
       出典:
       <a href="{source_url}">{source_url}</a>
     </p>
     ```
   - `WPClient.update_post_fields(post_id, content=new_content)` で反映
   - status=publish のまま

3. **受け入れ条件**:
   - source 特定できた記事は `acceptance_fact_check --post-id <id>` で **`source_reference_missing` yellow が解消**（green or 別軸 yellow のみ）
   - 特定できなかった記事は WP 更新せず、理由をレポートに記載
   - regression: 既に green の publish 記事が red/yellow に転落しないこと
   - publish 全量再監査で **red 0件維持**

### 実行コマンド例

```bash
# 5件それぞれ現状確認
for pid in 61754 61596 61598 61572 61600; do
  echo "=== $pid ==="
  python3 -m src.acceptance_fact_check --post-id $pid --json
done

# 反映後の確認
for pid in 61754 61596 61598 61572 61600; do
  python3 -m src.acceptance_fact_check --post-id $pid --json \
    | python3 -c "import json,sys; d=json.load(sys.stdin); d=d[0] if isinstance(d,list) else d; \
      print(f\"{d['post_id']}\t{d['result']}\t{len(d['findings'])} findings\")"
done

# publish 全量で red が出ていないこと
python3 -m src.acceptance_fact_check --status publish --since all --limit 50 --json \
  | python3 -c "import json,sys; data=json.load(sys.stdin); \
    reds=[d for d in data if d['result']=='red']; \
    print('red count:', len(reds))"
```

### 成功判定

- 5件のうち **最低 3件** で `source_reference_missing` yellow 解消
- source URL 特定できなかった記事は理由を記載（例: 類似記事がどの媒体にも見当たらない）
- regression なし（p=62527 / p=61981 / 他 publish の状態不変）
- publish red 0件維持

### 失敗時

- source URL 特定できる件数が 1 件以下 → backfill は諦めて除外ルール(c) 検討に切り替える旨をレポートに記載、WP 更新は行わず中止
- WP 更新時に update_post_fields が error を返す → 該当 post の状態を revert せず中断し、エラー内容をレポートに記載
- regression（他記事が red に転落）発生 → 即 WP 側を更新前に戻して中止
