## 依頼内容

**本便は観測性向上のみ**。実際の Gmail 側バグ（フィルタ・認証・DMARC 等）調査は本便の範囲外。

### 目的

- `send_email()` に以下の 3 情報を追加ロギング:
  1. `message_id`: `email.utils.make_msgid()` で明示生成
  2. `refused_recipients`: `smtp.send_message()` の戻り値
  3. `smtp_response`: 可能なら SMTP サーバの最終 response line
- 手動で 1 回だけ `/fact_check_notify` を叩き、新ログで refused 有無を観測
- refused が空なら SMTP 経路は健全 → Gmail 側調査（本便外）へ
- refused が非空なら原因が特定できる

### 要件

1. **`src/fact_check_notifier.py:384-416` の `send_email()` 修正**:
   ```python
   from email.utils import make_msgid

   def send_email(...) -> dict[str, Any]:
       password = _load_gmail_app_password()
       if not password:
           # 既存の demo モードはそのまま
           ...

       message_id = make_msgid(domain="yoshilover.com")
       msg = EmailMessage()
       msg["Subject"] = subject
       msg["From"] = from_email
       msg["To"] = to_email
       msg["Message-ID"] = message_id
       msg.set_content(text_body)
       msg.add_alternative(html_body, subtype="html")

       host, port = _smtp_config()
       with smtplib.SMTP_SSL(host, port, timeout=20) as smtp:
           smtp.login(from_email, password)
           refused = smtp.send_message(msg)  # dict[str, tuple[int, bytes]]
       return {
           "mode": "smtp",
           "subject": subject,
           "to_email": to_email,
           "from_email": from_email,
           "message_id": message_id,
           "refused_recipients": {k: [v[0], v[1].decode("utf-8", errors="replace")] for k, v in (refused or {}).items()},
       }
   ```

2. **`run_notification()` の `fact_check_email_sent` ログに追加**:
   - `message_id=delivery.get("message_id")`
   - `refused_recipients=delivery.get("refused_recipients")` (空 dict でも記録)

3. **regression test 追加**（`tests/test_fact_check_notifier.py` or 既存 test ファイル）:
   - `send_email()` が password ありの場合 `message_id` を含む戻り値を返すこと
   - `refused_recipients` が空 dict のときログが空 dict で出ること（mock smtplib）

4. **deploy + 手動 trigger**:
   - commit → push → Cloud Run deploy（`git archive HEAD` 経由、第11便と同手順）
   - 新 revision で `/fact_check_notify?since=yesterday` を 1 回だけ curl トリガー
   - Cloud Logging で `fact_check_email_sent` の `message_id` / `refused_recipients` を確認
   - 同 Message-ID が Gmail に届いてるかはよしひろさん側で確認（`rfc822msgid:<message-id>` で Gmail 検索）
