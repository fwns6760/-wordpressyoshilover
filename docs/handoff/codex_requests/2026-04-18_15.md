## 依頼内容

### 目的

`fact-check-morning-report` の cron 式を `0 7,12,17,22 * * *` → `*/30 * * * *`（毎30分）に変更。

### 要件

1. **Scheduler 変更（1 コマンドのみ）**:
   ```bash
   gcloud scheduler jobs update http fact-check-morning-report \
     --location asia-northeast1 \
     --project baseballsite \
     --schedule '*/30 * * * *' \
     --time-zone 'Asia/Tokyo'
   ```

2. **変更後確認**:
   ```bash
   gcloud scheduler jobs describe fact-check-morning-report \
     --location asia-northeast1 \
     --project baseballsite \
     --format='value(schedule,timeZone,state)'
   ```

3. **次の自然発火を 1 回観測**（次の :00 / :30 JST）:
   - `fact_check_email_sent` ログが出ること
   - `message_id` / `refused_recipients={}` / `smtp_response=[250, ...]` が継続して健全であること

4. **env 変更しない**:
   - `RUN_DRAFT_ONLY=1` / `AUTO_TWEET_ENABLED=0` / `PUBLISH_REQUIRE_IMAGE=1` 維持

5. **コード変更しない**（本便は Scheduler のみ）

### 期待される出力

```
schedule: */30 * * * *
timeZone: Asia/Tokyo
state: ENABLED
```

### 成功判定

- `gcloud scheduler jobs describe` で `schedule=*/30 * * * *` / `timeZone=Asia/Tokyo` / `state=ENABLED`
- 次の :00 or :30 JST 発火で `fact_check_email_sent` イベントが観測される
- `refused_recipients={}` 継続

### 失敗時

- update コマンドエラー → メッセージをレポートに記載、変更前のスケジュールに戻す（rollback）
  ```bash
  gcloud scheduler jobs update http fact-check-morning-report \
    --location asia-northeast1 \
    --project baseballsite \
    --schedule '0 7,12,17,22 * * *' \
    --time-zone 'Asia/Tokyo'
  ```

### 禁止

- env 変更
- コード変更
- Secret Manager 触らない
- 他 Scheduler ジョブのスケジュール変更
- Phase C 公開解放
- デプロイ
