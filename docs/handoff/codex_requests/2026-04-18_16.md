## 依頼内容

### 目的

- fact-check メール本文冒頭に「📊 直近 24h 運用サマリ」セクションを追加
- 既存の Cloud Logging イベント `rss_fetcher_run_summary` を集計する
- セクション追加のみ。既存の 3 セクション（🔴/🟡/✅）や auto-fix サマリはそのまま

### 要件

1. **集計ロジック**（`src/fact_check_notifier.py` に helper 追加）:
   - Cloud Logging client で直近 24h の `resource.type="cloud_run_revision" AND jsonPayload.event="rss_fetcher_run_summary"` を取得
   - 以下を集計:
     - `drafts_created` 合計
     - `created_subtype_counts` の merge（subtype 別内訳）
     - `skip_duplicate` / `skip_filter` / `error_count` 合計
     - `x_post_count` 合計
     - publish カウント: 既存 log に `publish_count` / `published_subtype_counts` 相当があれば採用、なければ「0 件（Phase C 未解放）」表示
   - 例外時は「集計取得失敗」セクションで失敗理由のみ表示（メール全体は止めない）

2. **メール本文への埋め込み**:
   - text body と html body 両方にセクション追加
   - 位置: サマリ（`🔴0件 / 🟡0件 / ✅0件 / checked=N`）の直後、🔴セクション前
   - フォーマット例:
     ```
     📊 直近24h運用サマリ
     ─────────────
     📥 作成: 12件（lineup=5, postgame=3, manager=2, player=2）
     📤 公開: 0件（Phase C 未解放）
     🔄 スキップ: 重複=32 / フィルタ=99 / エラー=0
     🐦 Xポスト: 10件
     ```

3. **既存コードに影響与えない**:
   - 他セクションのレイアウト維持
   - メール送信成否は既存判定のまま
   - サマリ取得失敗しても `fact_check_email_sent` は出す

4. **test 追加**（`tests/test_fact_check_notifier.py`）:
   - `rss_fetcher_run_summary` を mock して集計ロジックが期待通り動くこと
   - Cloud Logging 取得失敗時に「取得失敗」表示でメール本体が出ること

5. **deploy 後確認**:
   - `git archive HEAD | tar -x` 方式で Cloud Run source deploy
   - smoke test + scheduler 手動 trigger 1 回
   - メール本文に新セクションが含まれることを Cloud Logging `fact_check_email_sent` の `text_preview` で確認
   - よしひろさん Gmail 側でも次の :00 or :30 自然発火で確認（30分おき稼働中）

### 実行コマンド例

```bash
# 1. test
python3 -m unittest discover -s tests

# 2. deploy
tmp=$(mktemp -d)
git archive HEAD | tar -x -C "$tmp"
gcloud run deploy yoshilover-fetcher \
  --source "$tmp" \
  --region asia-northeast1 \
  --project baseballsite

bash scripts/cloud_run_smoke_test.sh <new-revision>

# 3. 手動 trigger
gcloud scheduler jobs run fact-check-morning-report \
  --location asia-northeast1 --project baseballsite

# 4. ログ確認
gcloud logging read \
  'resource.type=cloud_run_revision AND jsonPayload.event="fact_check_email_sent"' \
  --limit 1 --project baseballsite --format json \
  | jq '.[0].jsonPayload.text_preview'
```

- Cloud Logging 取得 permission エラー → 取得失敗 fallback でメール自体は送る

- deploy smoke test 異常 → 前 revision に rollback

### 禁止

- env 変更
- Scheduler スケジュール変更
- Secret Manager 触らない
- Phase C 公開解放
- 既存 3 セクション（🔴/🟡/✅）のレイアウト変更
- subject 変更
