# Codex 依頼書 — 2026-04-18（第17便・T-001 yoshilover-exclude-cat.php の REST_REQUEST 除外）

**依頼者**: よしひろさん

**本便の目的**: `src/yoshilover-exclude-cat.php` の 2 フック（`pre_get_posts` / `posts_where`）を **REST リクエスト中は早期 return** するよう修正し、T-001（WP REST API の `status=draft` フィルタが `publish` 固定に書き換わる問題）を解消

## 背景

- T-001 真犯人: `src/yoshilover-exclude-cat.php` の `pre_get_posts` フック内で `$query->set('post_status', 'publish')` している
- `is_admin()` で早期 return しているが **REST リクエストでは `is_admin()=false`**
- そのため REST collection query（`/wp/v2/posts?status=draft`）でも `post_status=publish` に書き換えられ、draft が取れない
- 既に原因特定済み（OPEN.md T-001 記載）、修正方針も決定済み

**方針確定**:
- 両フック（`pre_get_posts` / `posts_where`）の先頭に `if ( defined('REST_REQUEST') && REST_REQUEST ) return; ` を追加
- REST collection query はこのプラグインから除外する（REST は元々 admin 相当のコンテキストで使われる想定、front-end 除外対象外）

## 本便スコープ

- **コード修正 + local syntax check + commit** まで（**WP への deploy 含まない**）
- WP (Xserver) への差し替えはよしひろさん手動 or 第18便で SSH 便として切り出す
- Cloud Run / Cloud Scheduler / Secret Manager には一切触らない
