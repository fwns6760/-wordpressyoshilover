### Step 4: 薄さの定量化

各記事について以下を集計して表に:

| post_id | subtype | source | title 文字数 | 本文 HTML 文字数 | 本文 プレーン文字数 | 段落 `<p>` 数 | `<h2>/<h3>` 数 | 画像 `<img>` 数 | 出典 `<a>` 数 | AI 段落の文字数 |
|---|---|---|---|---|---|---|---|---|---|---|

集計ノート（主観も含めて OK、Claude Code が判定する）:

- 「薄い」と感じる観点の候補（response doc に **観察コメント**として記載）:
  - 元ソースの引用が少ない / 要約だけ
  - 段落構成が単調（見出しなし、箇条書きなし）
  - カテゴリ特有の観点（postgame なら「試合の流れ」「次戦への含み」、lineup なら「起用意図」）が無い
  - 数字が出てこない（打率・防御率・順位・対戦成績）
  - 読者視点の問いかけ / 解釈が無い
- **ヨシラバー × AI の掛け算観点**（重点評価）:
  - この記事は **他サイトの要約と何が違うか**（差別化点が 1 つでも書かれているか）
  - **過去の文脈**（過去試合・過去発言・選手経歴）への接続があるか
  - **複数視点**（選手・監督・ファン・データ）の対比があるか
  - **数字と物語の両輪**（統計だけ / 情緒だけに偏っていないか）
  - **AI ならではの広域参照**（シーズン序盤からの流れ、他球団比較、歴史比較）があるか
  - もし全部「No」なら、それは「AIで要約」であって「ヨシラバーで書く意味」が弱い、の根拠になる

### Step 5: subtype × 期待構成 のマッピング提案（ヨシラバー × AI 視点）

コード修正ではなく、「こう変えたい」の叩き台を response doc に書く。
**各 subtype で "ヨシラバーだから書ける 1 行" と "AI だから可能な 1 行" を必ず入れる**:

- **postgame**:
  - 試合結果 → キーマン → 次戦展望 → 読者への問い（基本構成）
  - *ヨシラバー固有*: 過去の同カード / 同投手との比較、選手の今季の流れ
  - *AI 活用*: シーズン開幕からの推移、対戦成績、打率・防御率の即時引用
- **lineup**:
  - 発表スタメン → 起用意図推測 → 打線バランス観察
  - *ヨシラバー固有*: 直近 3〜5 試合との差分（誰が外れた / 入った、理由仮説）
  - *AI 活用*: 相手先発との相性、過去対戦データ
- **manager/notice**:
  - コメント原文 → 背景 → 読者論点
  - *ヨシラバー固有*: 阿部監督のこれまでの発言トーンと今回の差
  - *AI 活用*: 過去の類似コメントとの比較、タイミング的意味
- **farm**:
  - 選手名・成績 → 一軍昇格可能性
  - *ヨシラバー固有*: 「なぜ今このタイミングで注目か」の文脈
  - *AI 活用*: 他球団の同ポジション若手との比較、二軍成績の推移
- **pregame/recovery/social/player/general**: 同様に「固有視点 1 行 + AI 1 行」を必ず含める

提案は **1 subtype あたり 5〜8 行** でよい。過剰設計にしない、しかし **「他サイトの要約と何が違うか」が 1 行で言える**ことが必須条件。

### Step 6: 結論と次便候補

response doc に以下を記載:

1. **現行 prompt の全体評価**（1 段落）: 構造 / 指示の強さ / subtype 分岐の深さ / **ヨシラバー × AI の掛け算が織り込まれているか**
2. **薄さの主因 top 3**（番号付き、根拠として表を引用）。必ず **「他サイトとの差別化できているか」** の観点を 1 つ含める
3. **次便候補**:
   - (A) prompt rule の段階拡張（subtype 別、1〜2 subtype ずつ、A/B 観察可）
   - (B) build_news_block() の HTML テンプレ側に「構成セクション」導入（固有視点セクション / AI 広域参照セクション）
   - (C) 元ソースの引用範囲拡大（`summary` より深く抜粋）
   - (D) **過去記事への参照導線**（同選手 / 同カードの過去記事タグ / internal link 自動挿入）
   - (E) **数値・データソース** の自動付与（選手成績 API / 対戦成績 等 — 可能性検討）
   - (F) その他
4. **優先度推奨**: Claude Code が判定しやすいよう、Codex 視点で 1 つに絞って推奨（理由 2〜3 行）。**"ヨシラバーで書く意味" を最も増やせる候補**を選ぶ
