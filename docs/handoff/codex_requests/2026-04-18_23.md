**本便の目的**: **週末 18:00-19:59 JST の `/run` scheduler gap を埋める。** 平日 filter/policy 問題は触らない。

## 背景

- 第22便で、`18時から記事が作成されない` 現象は 1 つではないと判明
- **2026-04-18(土)** の 18:00 / 18:30 / 19:00 / 19:30 JST は、`/run` request 自体が存在しない
- 原因は scheduler 構成:
  - `giants-weekend-post`: `0,30 16,17 * * 0,6`
  - `giants-weekend-eve`: `0,30 20-22 * * 0,6`
- したがって **週末 18:00-19:59 JST が空白**
- 一方で平日 18-20 時は `/run` が発火しており、そちらは `pregame_started` / `live_update_disabled` / social filter の議論

## 本便スコープ

- Cloud Scheduler のみ変更
- 推奨修正: `giants-weekend-eve` を **`0,30 18-22 * * 0,6`** に更新
- 調査や docs だけで終わらせず、**変更 → describe 確認 → 手動 trigger 1 回 → response doc → commit/push** までやる

## 禁止

- コード変更
- Cloud Run deploy
- env / Secret / revision 変更
- 平日 scheduler の変更
- `--set-env-vars` など service 全量上書き系コマンド

## Step 5: rollback 手順

異常時はこれで元に戻す:

```bash
gcloud scheduler jobs update http giants-weekend-eve \
  --location asia-northeast1 --project baseballsite \
  --schedule "0,30 20-22 * * 0,6" \
  --time-zone "Asia/Tokyo"
```

rollback 後に describe:

```bash
gcloud scheduler jobs describe giants-weekend-eve \
  --location asia-northeast1 --project baseballsite
```

### rollback 条件

- 手動 trigger 直後に `/run` request が失敗
- `severity>=ERROR` が増えた
- 想定外カテゴリの publish / 振る舞い異常が見えた
