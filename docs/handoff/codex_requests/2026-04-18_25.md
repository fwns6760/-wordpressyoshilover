
## Step 3: subtype × skip_reason のクロス表

### コマンド例

```bash
# Python で cross tab（jq より安定）
python3 << 'EOF'
import json, collections, datetime
with open('/tmp/flow_7d.json') as f:
    data = json.load(f)
cnt = collections.Counter()
for ent in data:
    ts = ent.get('timestamp','')
    try:
        t = datetime.datetime.fromisoformat(ts.replace('Z','+00:00'))
    except Exception:
        continue
    jst = t + datetime.timedelta(hours=9)
    if jst.weekday() >= 5:   # 0=Mon..6=Sun
        continue
    if jst.hour not in (18, 19):
        continue
    p = ent.get('jsonPayload', {})
    prep = p.get('prepared_category_counts', {}) or {}
    skip = p.get('skip_reasons', {}) or {}
    for sub, n in prep.items():
        cnt[('prepared', sub)] += n
    for r, n in skip.items():
        cnt[('skip', r)] += n
    for sub, n in (p.get('created_subtype_counts', {}) or {}).items():
        cnt[('created', sub)] += n
print('\n'.join(f'{k[0]}\t{k[1]}\t{v}' for k,v in cnt.most_common()))
EOF
```

### 期待出力

```
prepared	pregame	28
skip	pregame_started	22
created	lineup	9
...
```

### 成功判定

- prepared / skip / created の 3 view が同じ subtype 軸で比較できる
- 「prepared は多いが skip で大半落ちる」subtype が identify される

### 失敗時

- python 実行環境なし → `python3 -m venv` or system python3 を使用。`.env` は不要（ログデータのみ参照）

### 所要時間

- 8 分
