# Codex 依頼書 — 2026-04-18（第27便・T-021 過去記事リンク自動挿入の実装）

## 目的

記事薄さ対策として、公開済み過去記事へのリンクを生成本文に自動挿入する。「ヨシラバーだから書ける」軸を強化する機能。

## 対象

- `src/rss_fetcher.py` の本文生成経路（`build_news_block` 後段）
- `tests/` に unit test 追加
- 本便は code + test まで。**deploy は次便に分離**

## 仕様

- 生成本文の末尾、CTA 手前に `【関連記事】` セクションを挿入
- 関連記事の選定基準（優先順位）:
  1. タイトルに同一選手名を含む公開済記事（30 日以内、新しい順）
  2. 同 category + 同 subtype の公開済記事（30 日以内、新しい順）
  3. 同 category の公開済記事（30 日以内、新しい順）
- 自記事を除外、最大 2 件
- 候補 0 件ならセクション自体を省略（空見出しを出さない）
- WP REST の `/wp/v2/posts?search=<player>&status=publish&per_page=5` 等で取得、失敗時は 0 件扱い（記事生成は止めない）
- 選手名抽出は既存の subject 抽出ロジックを流用（新規実装しない）
- 挿入する HTML は既存のカード/CTA と干渉しないよう独立 `<div class="yoshilover-related-posts">` で囲む
