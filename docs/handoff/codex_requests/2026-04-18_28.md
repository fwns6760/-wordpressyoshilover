# Codex 依頼書 — 2026-04-18（第28便・T-021 実装の deploy + smoke）

## 目的

第27便 `9ae1169` で実装した「過去記事リンク自動挿入」を Cloud Run `yoshilover-fetcher` へ deploy し、実公開記事で `【関連記事】` が出ることを smoke 確認する。

## 対象

- Cloud Run service: `yoshilover-fetcher`（project `baseballsite` / region `asia-northeast1`）
- Deploy 元: 現 master（`9ae1169` を含む HEAD）
- 既存 env / Secret / scheduler は **不変**

## やること

- 新 revision を build + deploy（既存の deploy 手順に従う。`--set-env-vars` など全量上書き禁止、`--update-env-vars` も本便は使わない）
- deploy 後 `gcloud run services describe yoshilover-fetcher --region asia-northeast1 --project baseballsite` で新 revision 名と traffic 100% を確認
- `gcloud scheduler jobs run giants-weekend-eve --location asia-northeast1 --project baseballsite` で手動 trigger 1 回
- Cloud Run log で `/run` HTTP 200 / `rss_fetcher_run_summary` の `drafts_created` / `publish_count` / `error_count` を記録
- 公開された記事が 1 本以上あれば、そのうち 1 本を WP REST `GET /wp/v2/posts/<id>?context=edit` で取得し、本文に `<div class="yoshilover-related-posts">` と `【関連記事】` が含まれているか確認（候補 0 件で省略された場合はその旨を明記）
- 異常あれば rollback（1 つ前の revision に traffic 100%）
