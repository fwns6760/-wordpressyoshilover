## 仕様

### A. 新ヘルパ `_format_team_batting_stats_block(team_stats: dict) -> str`

- 配置: `fetch_giants_batting_stats_from_yahoo` の直上 or 直下（`rss_fetcher.py` 内、近接セクション）
- 入力: `{選手名: {"打率": str, "本塁打": str|int, "打点": str|int, ...}}` 形式（Yahoo 側の既存 dict 構造に合わせる。キー名は既存 `_parse_yahoo_team_batting_stats` の返り値を読み取って合わせること。**新規キー名の発明は禁止**）
- 出力: 上位最大 5 行のプレーンテキスト。並び順は「打率（float 変換可能なものだけ）降順 → 打点降順 → 選手名昇順」。欠損値は `-` で描画。
  ```
  ・坂本勇人 .312 / 5本 / 23打点
  ・岡本和真 .298 / 8本 / 25打点
  ・...
  ```
- 入力が空 dict / None / Exception（float 変換失敗を含む）→ `""` を返す。**生成処理を止めない**
- 末尾に改行は付けない（呼び出し側で囲い込む）

### B. strict prompt 引数追加

以下 3 関数に `team_stats_block: str = ""` を追加（位置引数の最後、デフォルト付きのまま。既存呼び出しの後方互換を壊さないこと）:

- `_build_gemini_strict_prompt`
- `_build_game_strict_prompt`
- `_build_manager_strict_prompt`

各関数内、`team_stats_block` が空文字でないときだけ、既存の `source_fact_block` ブロックの **直後** に以下を挿入:

```
【参考：巨人打者の今季主要指標（下の数字そのものを本文に転記する場合は source facts と矛盾しないことを確認）】
{team_stats_block}
・上記は参考値。source facts と食い違う場合は source facts を優先する
・個別選手の成績を本文に書くときは .xxx / 本 / 打点 の3点セットで並べる（他指標を勝手に足さない）
```

空文字時は一切追加しない（既存 prompt と完全同一であること。回帰テストで担保）。

`_build_gemini_strict_prompt` 内の分岐（`_build_game_strict_prompt` / `_build_manager_strict_prompt` への委譲）で、受け取った `team_stats_block` をそのまま下流へ転送。選手情報 / 首脳陣以外のカテゴリ（ドラフト・育成、社会系）では **渡さない**（空文字のまま）。

### C. 呼び出し側（`generate_article_with_gemini` 内、strict_mode 分岐）

- env flag: `ARTICLE_INJECT_TEAM_STATS`（既定 `"0"`）
- `os.environ.get("ARTICLE_INJECT_TEAM_STATS", "0") == "1"` かつ `category in ("試合速報", "首脳陣")` の時だけ:
  ```python
  team_stats_block = ""
  try:
      team_stats = fetch_giants_batting_stats_from_yahoo()
      team_stats_block = _format_team_batting_stats_block(team_stats)
  except Exception as e:
      logging.getLogger("rss_fetcher").warning("team batting stats 取得失敗: %s", e)
      team_stats_block = ""
  ```
- その値を `_build_gemini_strict_prompt(..., team_stats_block=team_stats_block)` に渡す
- flag OFF or 他カテゴリ → `team_stats_block=""` のまま（現行動作と 1 byte 違わない）

### D. ログ

`team_stats_block` が空でない場合のみ、prompt 組み立て直後に 1 回:

```python
logger.info(json.dumps({
    "event": "article_team_stats_injected",
    "category": category,
    "line_count": team_stats_block.count("\n") + 1,
}, ensure_ascii=False))
```

空の時はログ出さない（ノイズ防止）。

## テスト

`tests/` 配下の既存構成に合わせて unit test を追加（新規ファイルでも既存 test ファイルに追記でも可）:

1. `_format_team_batting_stats_block`:
   - 正常: 6 選手 dict を渡し、降順 5 行になること
   - 空 dict → `""`
   - `None` → `""`
   - 打率が `"-"` / 非数 混在 → 非数は末尾に回り、クラッシュしない
2. `_build_gemini_strict_prompt`:
   - `team_stats_block=""` での出力が、追加前の出力と **文字列一致**（回帰）

   - 同じ block を渡しても `category="ドラフト・育成"` の委譲先では含まれない（`_build_farm_strict_prompt` には引数追加しない、という仕様の担保）
3. `generate_article_with_gemini` 呼び出し側は network を叩くのでテスト対象外。flag OFF 時に `fetch_giants_batting_stats_from_yahoo` が呼ばれないことだけ `monkeypatch` で担保
