# Codex 依頼書 — 2026-04-18（第31便・T-021 第2弾 deploy + flag OFF smoke）

## 目的

第30便 `b7f20d8` で実装した「team batting stats の opt-in 注入（flag 既定 OFF）」を Cloud Run `yoshilover-fetcher` に deploy する。**flag OFF のまま** revision を切り替え、既存挙動に回帰がないこと（drafts_created / publish_count / error_count が通常レンジ、既存テンプレの本文生成が壊れていないこと）を smoke で確認する。flag ON 検証は別便。

## 対象

- Cloud Run service: `yoshilover-fetcher`（project `baseballsite` / region `asia-northeast1`）
- Deploy 元: 現 master（`b7f20d8` を含む HEAD）
- env / Secret / scheduler は **不変**。特に `ARTICLE_INJECT_TEAM_STATS` は **設定しない**（env に存在しない = 既定 "0" = OFF）
- 第28便と同じ deploy 手順

## やること

- 新 revision を build + deploy
  - `--set-env-vars` 全量上書き禁止
  - `--update-env-vars` は本便で使わない（新 flag を追加しない）
  - deploy コマンドで env を一切触らない（既存 env が維持されていることを describe で確認）
- `gcloud run services describe yoshilover-fetcher --region asia-northeast1 --project baseballsite` で:
  - 新 revision 名
  - traffic 100% が新 revision に向いていること
  - env に `ARTICLE_INJECT_TEAM_STATS` が **存在しないこと**（第2弾が実質 OFF であることの担保）
  - 既存 env（`GMAIL_APP_PASSWORD` 参照、`GEMINI_FLASH_MODEL` など、第28便時点の一覧）が変わっていないこと
- `gcloud scheduler jobs run giants-weekend-eve --location asia-northeast1 --project baseballsite` で手動 trigger 1 回
- Cloud Run log で以下を記録:
  - `/run` HTTP 200
  - `rss_fetcher_run_summary` の `drafts_created` / `publish_count` / `error_count`
  - `article_team_stats_injected` イベントが **0 件であること**（flag OFF の担保 — 出ていたら即 rollback）
  - 既存の `article_ai_route_override` / `gemini_strict_*` 系ログ出現頻度が第28便時点と同レンジであること
- もし draft が 1 本以上生成されたら、そのうち 1 本を WP REST `GET /wp/v2/posts/<id>?context=edit` で取得し:
  - 本文末尾に第1弾の `<div class="yoshilover-related-posts">` が候補あり時のみ出ていること（第2弾によって壊れていないこと）
  - 本文中に「【参考：巨人打者の今季主要指標」の文字列が **含まれないこと**（flag OFF の担保）
- 異常（`error_count > 0` が第28便時点より有意に増、`article_team_stats_injected` が1件でも出る、既存テンプレ文言欠落、describe の env 破壊）があれば即 rollback（1 つ前の revision に traffic 100%）し、rollback 後 describe まで取得
