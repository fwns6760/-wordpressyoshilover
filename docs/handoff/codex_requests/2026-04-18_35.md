## やること

### 1. `/audit_notify` endpoint 新設（`src/server.py`）

- HTTP method: `GET /audit_notify`
- 認証: `X-Secret: $RUN_SECRET`（既存 `/fact_check_notify` と**同 secret 再利用**、新規作成禁止）
- query parameter:
  - `window_minutes` (optional, default `60`): 監査対象の過去 N 分 window
- 監査対象:
  - WordPress posts、status = `draft` + `publish` **両方**
  - window 内に **作成 or 更新された記事**
- 監査軸（**4軸+pipeline**）:
  1. `title_body_mismatch` — title に出てくる事実と本文の整合性（既存 acceptance_fact_check の軸流用で可）
  2. `thin_body` — **`<div class="yoshilover-related-posts">` を除外したコア本文の実文字数**で判定。閾値: `postgame`/`lineup` = 280、`column`/`manager` = 350
  3. `no_opinion` — 主観マーカー語ベース（ルール）。`AUDIT_OPINION_USE_LLM` env で LLM 切替可だが **既定 OFF**、本便はルールのみ動作確認
  4. `no_eyecatch` — WP post の `featured_media` 欠損
  5. `pipeline_error` — 既存 fact check の `error_count` / 投稿失敗系イベントを流用
- **`repetition` 軸は対象外**（別便で起票予定、本便スコープ外）
- 戻り値 schema（JSON）:
  ```json
  {
    "window_minutes": 60,
    "counts": {
      "title_body_mismatch": 0,
      "thin_body": 0,
      "no_opinion": 0,
      "no_eyecatch": 0,
      "pipeline_error": 0
    },
    "total": 0,
    "mail_sent": false,
    "findings": [
      {
        "post_id": 12345,
        "title": "...",
        "status": "draft",
        "primary_category": "postgame",
        "article_subtype": "...",
        "edit_url": "https://.../wp-admin/post.php?post=12345&action=edit",
        "axis": "thin_body",
        "excerpt": "（先頭200字）"
      }
    ]
  }
  ```
- 0件時: **メール送信スキップ**（`mail_sent: false`、HTTP 200 で返す）
- 非0件時: `send_email()` (`src/fact_check_notifier.py:713`) 経由で以下のメールを送信 → `mail_sent: true`

### 2. メール format

- 送信先: `fwns6760@gmail.com`（`src/fact_check_notifier.py:34` の既定値再利用）
- **件名**:

