# 2026-04-20_79_wp_draft_reuse_bug_fix

## 目的

`wp_client.find_recent_post_by_title()` の title-only + 24h 再利用が、generic rewritten title と組み合わさって distinct source を同一 draft に誤吸着する bug を修正する。直近 7 日で 46 fire / 248 fire = 18.5% の汚染率（便 #78 診断）。

## 対象ファイル / 対象リソース

- `src/wp_client.py` — `find_recent_post_by_title()` / `create_post()` / `_reuse_existing_post()`
- `src/rss_fetcher.py` — fire ループ内 same-fire guard 追加
- `tests/test_wp_client.py` — reuse 条件テスト追加
- `tests/test_duplicate_prevention_golden.py` — same-fire guard golden test 追加
- `docs/handoff/tickets/OPEN.md` — 本バグのチケット追記（ID: `T-025`）

## やること

- `src/wp_client.py` 新規 draft 作成時に custom meta `_yoshilover_source_url` を保存する。既存 meta key が別名で実在する場合はそれを再利用する
- `find_recent_post_by_title()` のシグネチャに `source_url: Optional[str] = None`, `allow_title_only_reuse: bool = False` を追加
- `find_recent_post_by_title()` の reuse 判定を下記に変更:
  - `source_url` が渡され、既存 post の `_yoshilover_source_url` meta と完全一致 → reuse 許可
  - `allow_title_only_reuse=True` が渡された場合 → reuse 許可（retry / 手動復旧用）
  - それ以外（title 一致のみ、source_url 不一致 or meta 未設定） → **reuse しない、新規 draft 作成**
- `create_post()` の reuse パスに `source_url` を伝播
- reuse 時のログに `reuse_reason=source_url_match` または `reuse_reason=title_fallback` を付記（既存 `[WP] 既存記事を再利用 post_id=...` に追加）
- `_reuse_existing_post()` で既存 post に `_yoshilover_source_url` meta が未設定なら reuse 時に backfill 保存する
- `src/rss_fetcher.py` の fire ループ内で `same_fire_source_urls: set[str]` を持ち、同一 fire で distinct source_url かつ同一 rewritten title の場合は title-only reuse を抑止する（`allow_title_only_reuse=False` のまま、かつ source_url mismatch で新規作成される挙動に委ねる）
- distinct source 検出時のログ `same_fire_distinct_source_detected source_url=... rewritten_title=...` を emit
- `tests/test_wp_client.py` に unittest 4 本追加:
  - `source_url 一致時は既存 draft を reuse する`
  - `source_url 不一致かつ title 一致時は reuse せず新規作成する`
  - `allow_title_only_reuse=True 時は title 一致のみで reuse 可`
  - `新規 draft 作成時に _yoshilover_source_url meta が保存される`
- `tests/test_duplicate_prevention_golden.py` に unittest 3 本追加:
  - `同一 fire・別 farm URL・同一 generic title → post_id が分かれる`
  - `同一 fire・別 pregame URL・同一 generic title → post_id が分かれる`
  - `同一 source の再実行 retry で既存 draft reuse が維持される`
- `python3 -m unittest discover -s tests -v` で全 pass を確認
- `gcloud run deploy yoshilover-fetcher --project baseballsite --region asia-northeast1 --no-traffic --tag=dup-fix-canary` で canary deploy
- canary smoke を既存手順（一時 Cloud Scheduler job 経由）で 200 OK 確認
- prod 切替は user 判断のため、canary smoke 結果を Codex response に記載し、promote は **実施しない**（user go 待ち）
- `docs/handoff/tickets/OPEN.md` に `T-025: WP draft title-only reuse が distinct source を誤吸着（#78 診断、7日 18.5% 汚染、post_id 62795 11 fire 集中）` を追記
