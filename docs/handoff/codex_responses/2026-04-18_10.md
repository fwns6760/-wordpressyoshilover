### 実装方針

`extract_source_links()` に、調査便で確定した B/C 系の 4 パターンを追加した。

### 追加した抽出パターン

1. **P1: `【引用元】` 見出し + 次段落アンカー**
   - WordPress block comment (`<!-- wp:heading -->`, `<!-- wp:paragraph -->`) をまたいでも拾えるようにした

2. **P2: `引用元:` / `出典:` / `参考:` / `参照元:` / `source:` ラベル付き段落**
   - ラベルが見つかる `<p>` 内の anchor を抽出

3. **P3: 本文末尾の小文字 footer リンク**
   - `font-size` + `color:#666/#999` 系の `<p style=...>` に含まれる source URL を抽出

4. **P4: `[[1]](url)` 形式**
   - 本文中に埋まった markdown 風参照リンクを抽出

### 副作用防止

- `twitter.com` / `x.com` / `t.co` / `youtube.com` / `youtu.be` / `facebook.com` / `instagram.com` は除外
- footer / markdown 経路は既知 source ドメインに限定
- 既存の `📰 参照元:` 抽出は維持
- badge fallback (`📰 巨人公式X / ...`) も維持

### 実装 diff 要約

```diff
+ PARAGRAPH_RE
+ REF_HEADING_RE
+ MARKDOWN_LINK_RE
+ SOURCE_LABEL_RE
+ FOOTER_STYLE_RE
+ SOURCE_HOST_PATTERNS / EXCLUDED_SOURCE_HOST_PATTERNS
+ _is_supported_source_url()
+ _extract_anchor_items()
+ _append_unique_source_items()

- extract_source_links() を単一 regex -> 複数パターン走査へ拡張
```
