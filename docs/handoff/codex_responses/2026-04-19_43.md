# 2026-04-19_43

## Phase A: 設計判断

- 判断: `social_news` 由来でも、`article_subtype` が structured template を持つ `postgame / lineup / pregame / farm / farm_lineup` は、その subtype 側の template を優先する
- 維持: `manager / player / general social` は従来どおり social template を使う

### 根拠

- `62621` / `62598` の実ログでは、同一 run に `social_body_template_applied` と `game_body_template_applied` が同時に出ていた
- 旧実装は `source_type == "social_news"` なら prompt / safe fallback / publish gate を先に social 扱いしていた
- 一方で `category == "試合速報"` かつ game subtype のときは game template log も別で出しており、structured game template の意図自体は既にコード上に存在していた
- publish gate も `resolve_publish_gate_subtype()` が無条件で `social` を返していたため、`postgame` などに分類されても publish が social gate に吸われていた

## Phase B: 修正内容

### `src/rss_fetcher.py`

- `_social_source_prefers_structured_template()` を追加
  - `試合速報 + (postgame/lineup/pregame)`
  - `ドラフト・育成 + (farm/farm_lineup)`
  - の場合だけ social source でも structured template を優先
- `_build_gemini_strict_prompt()` の social 分岐を上記 helper 経由に変更
- `_build_safe_article_fallback()` の social 分岐も同様に変更
- `build_news_block()` の `social_story` 判定を helper 経由に変更
- `resolve_publish_gate_subtype()` を変更
  - social source でも structured game subtype はその subtype を返す
  - `farm / farm_lineup` は publish gate 上 `farm` に正規化
  - それ以外の social source は従来どおり `social`
- `social_body_template_applied` log を helper 経由に変更し、game/farm structured subtype では social log を出さないようにした

### social template 固定文言

- `_social_background_focus_line()` の `〜があれば必ず整理する` を、`source にある範囲だけを整理する` に変更
- `_build_social_safe_fallback()` の
  - `まず押さえておきたい話題です`
  - `どうつながるかが見どころです`
  を leak marker 非衝突の文言へ差し替え
