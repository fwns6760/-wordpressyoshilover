### X ポスト系が 39〜50 文字になる理由

| 段階 | 事実 |
|---|---|
| social feed 正規化 | `social_news` は `_clean_social_entry_text()` で URL / hashtag / mention / outlet suffix を除去する (`src/rss_fetcher.py:1479-1493`) |
| fact block の材料 | `_source_fact_block_metrics()` は title + summary しか見ない。リンク先本文や引用 RT 元は取得しない |
| fact block の構成 | `_extract_source_sentences(title, summary)` で短文を最大 8 個拾い、quote phrase を補足するだけ |
| 今回の 3 件 | cleaned title/summary だけでは 1〜2 文相当しか残らず、block 長が `39 / 45 / 50` に留まった |

### threshold `100` の根拠

| 根拠源 | 事実 |
|---|---|
| code / blame | `221e7f18` で hard-code された |
| test | `tests/test_publish_guard.py:67-84` は「短い source は constant 未満、長い source は constant 以上」を検証するのみで、`100` 自体の理由は持たない |
| commit message | `fix: add pre-publish quality guard`。数値理由の記述はない |
| handoff response | `docs/handoff/codex_responses/2026-04-19_39.md` に「source fact block 100 文字未満を skip」「将来 env 化可能」とある |

### 代替案比較

| 案 | 変える点 | 期待効果 | リスク / 未解決 |
|---|---|---|---|
| 案 1 | `social_news` だけ `min_chars=50` に緩める | 今回の `39/45/50` のうち `45/50` は draft 化可能になる | `39` は依然落ちる。短い X 投稿だけで draft 化される範囲が広がる |
| 案 2 | `source_fact_block` 抽出を増やし、X 本文や引用元を厚く取る | threshold を変えずに social source の通過率を上げられる | 取得元拡張の実装が必要。現コードには引用 RT 元やリンク先本文を fact block に入れる処理がない |
| 案 3 | 現状維持 | 短い X 単独ソースを確実に除外できる | 本日のように `manager/player social` は draft 0 のままになりうる |

## Phase C: 同便修正の可否

### 判定

- **進めない**

### 理由

1. root cause が単一ではない
   - `pregame_started = 6`
   - `thin_source_fact_block = 3`
2. `pregame_started` は `a3e6393e` の意図的ガードで、今回の 6 件は「試合開始後の pregame 扱い記事」として落ちている
3. `thin_source_fact_block` の修正には、threshold 緩和か source 拡張かの方針選択が必要
4. prompt 変更や publish gate 緩和を使わずに、同便で安全に 1 本に絞って直せる状態ではない
