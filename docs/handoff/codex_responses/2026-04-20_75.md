## 2. 実施内容

- `subtype 境界`
  `src/rss_fetcher.py` に `_has_lineup_core()` / `_has_live_update_fragment()` / `_has_explicit_confirmed_result()` を追加。`1番` と `3番手` を区別するために `番(?!手)` の正規表現へ切り替え、`継投 / 満塁 / 3者凡退 / サイクル安打王手 / N回 / N番手` を live update 断片として扱うようにした。
- `subtype 優先順位`
  `_detect_article_subtype()` を `明示的結果語 -> live_update 断片 -> score-only postgame -> lineup core -> pregame` に変更。これで lineup キーワードが混ざっても、試合進行中メモは `live_update`、`サヨナラ負け` のような結果確定文は `postgame` に倒れる。
- `title assembly`
  `巨人スタメン` 接頭辞の返却経路は既存どおり `subtype == "lineup"` 分岐のみ。漏れの主因は subtype 誤判定だったため、title builder 自体は増改築せず、reject 6件が `巨人スタメン` を返さないことを新規テストで固定した。
- `test`
  reject 6件それぞれについて `expected_subtype` と `expected_title` を追加。`62965 / 62962 / 62955 / 62948 / 62609` は `live_update`、`62621` は `postgame` として固定し、全件で `巨人スタメン` prefix 非付与を assert した。
