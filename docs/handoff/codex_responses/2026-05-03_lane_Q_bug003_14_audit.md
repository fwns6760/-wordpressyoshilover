- medium once authenticated WP revision / plugin / cron trail is available

Interpretation:

- it does not fully explain which actor made the currently field-observed `401` state persist on WP

### Rank 2. `B` `_reuse_existing_post()` silent upgrade and adjacent internal publish-bypass

Verdict:

- strongest repo-internal explanation for how `publish visibility` appeared without a guarded `sent` anchor

Evidence:

- `src/wp_client.py:_reuse_existing_post()` silently upgrades reused `draft/pending/future/auto-draft` posts to `publish` when caller requested publish
- `src/rss_fetcher.py:finalize_post_publication()` directly calls `wp.update_post_status(post_id, "publish")` outside guarded runner history / yellow / cleanup / postcheck
- 14/14 targets have `publish_time_iso`, but current-day guarded durable states are only `refused` or `skipped`
- 14/14 first non-sent observations occur `11s` to `5m10s` after `publish_at`, consistent with an unanchored publish being immediately re-evaluated by guarded logic

Reproducibility:

- high statically
- medium dynamically because this audit does not prove which exact caller hit `_reuse_existing_post()` for each post

Interpretation:

- `B` explains the ledger mismatch best
- `B` alone does not prove the final `401` actor, but it explains why posts can look publish-visible even though guarded durable state never lands on `sent`

### Rank 3. `A` cleanup mutation path

Verdict:

- weakest fit for this 14-id set

- `cleanup_failed_post_condition` is absent from the 14 current rows
- 14-id cleanup log hits are `0`
- 14-id yellow success log hits are `0`
- `_hold_reason_for_candidate_error()` only emits `cleanup_failed_post_condition` when `cleanup_required=true` and cleanup aborts; that shape is not what this 14-id cohort shows

Reproducibility:

- code path exists historically
- matching this 14-id signature is low

Interpretation:

- `cleanup_required=true` on backlog cluster means “yellow-origin post under reevaluation”, not proof of cleanup mutation
- current evidence does not support `A` as the immediate root cause
