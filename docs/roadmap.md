# Roadmap

## Phase C: 段階的公開

目的:

- subtype 単位で受け入れ試験しながら publish を解放する
- publish 安定後に X 投稿を別段階で解放する

完了条件:

- `postgame` / `lineup` / `manager` / `notice` を先行公開
- `pregame` / `recovery` / `farm` / `social` / `player` / `general` を順次判断
- `live_update` は引き続き publish / X とも別管理

## Phase D: アンテナサイト登録と SEO 強化

目的:

- 流入導線を増やす
- 巨人特化のトピッククラスターを強化する

候補:

- アンテナサイト登録
- カテゴリページ改善
- title / description / internal link の見直し
- Search Console / GA4 の観察導線整備

## Phase E: コンテンツ監査

目的:

- のもとけ型に寄せた在庫最適化
- 蓄積記事の評価と noindex 運用判断

時期:

- 6-12 ヶ月後

候補:

- 低品質 / 重複 / 流入ゼロ記事の抽出
- noindex 候補の棚卸し
- カテゴリ別の残す / 畳む判断

## Phase F: AI 時代の差別化

目的:

- 巨人特化の深掘りを AI 補助で継続自動化する
- 汎用まとめサイトと差別化する

候補:

- 選手 / 首脳陣 / 公示 / 二軍のニッチ深掘り自動化
- AI 向け引用されやすい要約構造の強化
- 反応ログと読者行動を使った記事改善ループ
- コメント / コミュニティ連携の検討
